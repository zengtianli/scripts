# 分层评分改造计划

## 背景

当前 Tab 2 权重计算后只有一个"试点整体年度评分"（C1-C10 直接加权求和）。用户需要**分层评分**：大循环、小循环、点循环各自有独立评分，最后三层面加权汇总得到试点总分。

## 核心逻辑

### 各层面评分
从 C1-C10 组合权重中提取各层面子集，**归一化为子权重**，然后对该层面指标 min-max 标准化后加权求和（百分制）：

- **大循环评分**：C1-C4 子权重归一化 → min-max → 加权求和 × 100
- **小循环评分**：C5-C6 子权重归一化 → min-max → 加权求和 × 100
- **点循环评分**：C7-C10 子权重归一化（企业均值数据）→ min-max → 加权求和 × 100

### 层面权重（自动推导）
各层面在 C1-C10 组合权重中的权重之和，即为该层面的权重：
- `w_大循环 = sum(w_C1..C4)`
- `w_小循环 = sum(w_C5..C6)`
- `w_点循环 = sum(w_C7..C10)`

三者之和自然为 1。

### 试点汇总评分
`试点总分 = 大循环评分 × w_大循环 + 小循环评分 × w_小循环 + 点循环评分 × w_点循环`

## 修改文件

### 1. `app.py` — Tab 2 "试点整体年度评分"区域（约 L221-252）

**替换**当前的单一评分逻辑为分层评分：

1. 定义层面分组：
   ```
   LAYER_GROUPS = {
       "大循环": ["C1", "C2", "C3", "C4"],
       "小循环": ["C5", "C6"],
       "点循环": ["C7", "C8", "C9", "C10"],
   }
   ```

2. 抽取通用函数 `_layer_score(data, cols, weights, directions)` → 百分制评分数组：
   - 从 indicator_cols 中找到属于该层面的列索引
   - 提取子权重并归一化
   - min-max 标准化（考虑正负向）
   - 加权求和 × 100

3. 展示：
   - 三列布局展示大循环/小循环/点循环各自的年度评分表
   - 展示层面权重
   - 展示试点汇总评分表（年度 | 大循环评分 | 小循环评分 | 点循环评分 | 汇总评分）
