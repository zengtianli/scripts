 # 浙东灌溉需水计算模型

## 1. 系统概述

### 1.1 业务背景与目标

浙东地区作为重要的农业生产区域，农业灌溉用水管理直接关系到粮食安全和水资源的合理配置。本模型旨在为浙东地区的农业灌溉管理提供科学的决策支持，通过精确计算未来一段时间内的灌溉需水量和排水量，实现：

- **水资源优化调度**：提前预知各分区的用水需求，合理调配水资源
- **灌溉计划制定**：为灌溉管理部门提供科学的灌溉计划
- **风险评估**：评估干旱/洪涝风险，提前采取应对措施
- **精细化管理**：支持多区域、多作物、多时段的精细化灌溉管理

### 1.2 管理覆盖范围

#### 1.2.1 服务区域

系统覆盖浙东地区15个重要平原区域，总管理面积超过4600平方公里，包括余姚平原水系（上河区、下河区、姚江上下游区、马渚中河区）、慈溪平原水系（西河区、中河区、东河区）和其他重要平原区（丰惠、南沙、绍虞、江北镇海、虞北上中河区、蜀山平原区）。农业生产管理涵盖单季稻377.7km²、双季稻566.5km²、旱地1136.3km²以及杂地1539.8km²和水面696.5km²，统筹考虑生态用水和农业用水需求。

#### 2.1.2 模块功能划分

- **main.py**：主程序入口，程序启动和执行控制
- **calculator.py**：主计算控制器，整合数据管理、灌溉管理、结果导出
- **dryland_models.py**：旱地作物模型，作物需水量计算
- **paddy_models.py**：水稻灌溉模型，水稻田水量平衡计算
- **utils.py**：工具函数集合，文件IO操作、数据处理和格式化
- **config.py**：全局配置参数，计算模式配置、文件路径设置
- **core.py**：模拟基类，所有模拟器的基础类

### 2.2 数据流与计算流程

```
1. main.py 启动程序
       ↓
2. config.py 加载配置
       ↓
3. calculator.py 初始化计算器
       ↓
4. utils.py 加载和处理输入数据
       ↓
5. 根据配置模式选择模拟器：
   - dryland_models.py (旱地作物模式)
   - paddy_models.py (水稻灌溉模式)
   - 或两者并行 (综合模式)
       ↓
6. utils.py 合并和导出结果
       ↓
7. main.py 完成执行
```

## 3. 核心业务功能

### 3.1 计算模式

系统支持三种计算模式：**作物需水模式**基于作物单位面积需水定额和种植面积进行理论需水量计算，支持75%和90%两种保证率水平，主要针对旱地作物；**灌溉系统模式**基于灌溉制度和水量平衡计算，模拟实际田间管理操作，适用于水田和旱地的实际灌溉模拟；**混合模式**结合两种模式的结果，提供更全面的灌溉需水评估。

### 3.2 水量平衡与决策支持

系统通过精准需水量计算，基于作物生长规律和生理需水特性，实时跟踪田间水位动态变化，综合考虑降雨补给、蒸发损失、土壤渗漏，精确计算每日灌溉需水量和排水量。智能灌溉决策支持包括根据作物类型制定差异化灌溉制度，设定科学的水位上下限管理标准，支持灌区分批次轮流灌溉，优化水资源配置。排水防涝管理通过精确计算农田排水需求，识别积水风险区域和时段，保障农作物免受涝害影响。

## 4. 关键业务概念

### 4.1 灌溉制度与水位控制

灌溉制度基于作物生长需求和田间水分管理要求，制定水位控制标准和灌溉操作规程。核心控制参数包括水位下限（H_min，-100mm到20mm）、水位上限（H_max，-88mm到40mm）和目标蓄水深度（Storage，0-80mm）。当水位低于H_min时进行灌溉补水到Storage水位，当水位高于H_max时进行排水降低到H_max水位，在H_min和H_max之间时保持现状。系统支持生育期差异化管理，根据作物不同生长阶段（育秧期、分蘖期、孕穗期等）动态调整蒸发系数（0.5-1.55）和水位控制参数。

### 4.2 轮灌制度与水量平衡

轮灌制度将灌区分成若干轮灌单元分批进行灌溉，合理分配有限水资源，减少供水设施压力，提高灌溉效率。水量平衡计算综合考虑降雨和灌溉作为输入项，蒸发蒸腾、渗漏和排水作为输出项，以田间水位深度作为状态变量，通过计算自然水位变化、渗漏损失和灌排需求，实现精确的水资源管理。

## 5. 数据来源与应用价值

### 5.1 数据处理

系统采用双重降雨数据源确保预报精度：前3天使用气象预报细化网格数据精确到15个河区，3天以后使用CFS系统中长期预报数据用于趋势判断。蒸发数据采用气象站观测的水面蒸发量，按河区配置逐日数据，通过蒸发折算系数转换为实际蒸发蒸腾量。

### 5.2 应用价值

系统服务于日常运行管理（制定7-30天灌溉计划、监控用水量、调度设施运行）、季节性管理（春灌准备、夏季防旱排涝、秋收保障）、应急管理（干旱洪涝快速响应）和规划决策（年度计划、工程评估、政策制定）四个层面。通过精准灌溉提升用水效率，避免水资源浪费；通过提前7-30天预测需水量增强防灾能力；推动农业灌溉数字化转型，支撑现代化发展。

## 6. 使用方法

### 6.1 基本操作

使用前准备输入文件（时间配置、分区信息、灌溉制度表、降雨蒸发数据、作物参数等），运行`python main.py`即可。系统支持三种计算模式：dryland（旱地作物）、paddy（水稻灌溉）、both（综合模式），可在config.py中配置。

### 7.2 输出文件说明

| 文件名             | 说明                     |
| ------------------ | ------------------------ |
| OUT_GGXS_C.txt     | 旱地作物模式的灌溉需水量 |
| OUT_GGXS_I.txt     | 水稻灌溉模式的灌溉需水量 |
| OUT_GGXS_TOTAL.txt | 合并后的总灌溉需水量     |
| OUT_PYCS_C.txt     | 旱地作物模式的排水量     |
| OUT_PYCS_I.txt     | 水稻灌溉模式的排水量     |
| OUT_PYCS_TOTAL.txt | 合并后的总排水量         |

### 7.3 使用方法与配置

**使用方法**

1. 准备输入文件，确保所有必要的数据文件都在同一目录下
2. 运行主程序：
   ```
   python main.py
   ```
3. 查看输出结果文件，分析灌溉需水量和排水量

**计算模式配置**

系统支持三种计算模式，可在 `config.py`中设置：

- **dryland**：仅计算旱地作物需水量
- **paddy**：仅计算水稻灌溉系统需水量
- **both**：同时计算旱地作物和水稻灌溉系统需水量，并合并结果

**重要配置选项**

在 `config.py`文件中可以修改以下重要配置：

- **CALCULATION_MODE**：计算模式（"dryland"、"paddy"或"both"）
- **USE_STATIC_IRRIGATION**：是否使用静态灌溉制度文件（true/false）
- **STATIC_IRRIGATION_FILES**：静态灌溉制度文件配置
- **IRRIGATION_SYSTEMS**：灌溉系统配置参数
- **OUTPUT_FILE_CONFIG**：输出文件配置

**输出文件路径**

程序运行后，所有输出文件会保存在 `./data/` 目录中，包括旱地作物模式、水稻灌溉模式的结果和合并结果。
