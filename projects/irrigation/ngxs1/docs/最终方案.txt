【确认理解】

    水稻模块：水量平衡法 → 不改，保持现状
    旱地模块：定额法     → 改，用流域中心新数据


【改动范围】

    只改旱地作物相关：

    ┌─────────────────────────────────────────────────────────────────────┐
    │                                                                     │
    │  xlsx_to_txt.py                                                     │
    │      读取 灌溉制度.xlsx → 生成 static_crops.txt（新格式）            │
    │                                                                     │
    │  dryland_models.py                                                  │
    │      修改 CropRegistry.load_from_file() 读取新格式                  │
    │                                                                     │
    │  config.py                                                          │
    │      INPUT_FILES['crop'] 指向新文件（如果文件名变）                  │
    │                                                                     │
    └─────────────────────────────────────────────────────────────────────┘


【不改】

    - 水稻相关全部不动
        - static_single_crop.txt  保留
        - static_double_crop.txt  保留
        - paddy_models.py         不改
    - 计算逻辑不动
    - 输出格式不动


【新旧数据格式对比】

    原 static_crops.txt：
        作物    75% mm/d    90% mm/d
        小麦    0.248       0.320
        ...
        （日需水强度，全年统一值）

    新 static_crops.txt（来自 灌溉制度.xlsx）：
        作物    保证率  1月  2月  3月  ...  12月  合计
        小麦    0.5     0    0    1   ...  0     5
        小麦    0.75    0    0    6   ...  0     40
        小麦    0.9     0    0    8   ...  0     50
        ...
        （月度定额，更精确）


【计算逻辑变化】

    原逻辑：
        日需水量 = 日需水强度(mm/d) × 面积
        （全年每天一样）

    新逻辑：
        日需水量 = 当月定额(m³/亩) ÷ 当月天数 × 面积
        （不同月份不同，更精确）

    ────────────────────────────────────────────────
    总计                                      ~135行


【使用流程】

    1. 修改 灌溉制度.xlsx（如需要）
    2. python xlsx_to_txt.py       # 只更新旱地作物数据
    3. python main.py              # 正常运行
