# 纳污能力计算 - 支流功能迁移方案

## 一、我的理解

### 现状
- **Streamlit 应用**（`web_app.py` + `calc_core.py`）目前只支持**单链干流**计算
  - 每个功能区只有一条河段，无支流概念
  - 功能区之间通过出流浓度链式传递
  - 输入格式：`功能区基础信息` sheet（每个功能区一行）+ `逐日流量` sheet（每个功能区一列）

- **VBA 版本**（`hd_计算.bas`）已实现完整的**干支流分段计算**：
  - 每个功能区可有多条支流
  - 支流在干流的指定位置汇入，将干流分成多段
  - 混合点按流量加权计算混合浓度
  - 只汇总干流段的纳污能力（支流自身不计入）

### 输入 xlsx 格式（`0河道功能区输入0122.xlsx`）

**`河道功能区-输入` sheet 布局**（竖向参数表，每个功能区一列）：

| 行 | 参数 | 示例值 |
|----|------|--------|
| 1 | 功能区数量 / 功能区编号 | 9 / 钱塘156, 钱塘157... |
| 2 | 方案个数 / 功能区名 | 1 / 钱塘156, 钱塘157... |
| 3 | Cs | 0.03 |
| 4 | K(1/s) | 2.315E-06 |
| 5 | b | 0.2~0.6 |
| 6 | a | 0.3~0.35 |
| 7 | β | 0.3~0.5 |
| 8 | 干流总长L(m) | 4000~15300 |
| 9 | 干流C0 | 0.02 |
| 10 | 支流数量 | 1 |
| 11 | 干流名 | 钱塘156-0 |
| 12+ | 支流信息（每条5行） | 名称、长度、汇入位置、浓度 |

**`逐日流量` sheet**：日期 + 每个功能区的干流和支流各一列（如 `钱塘156-0`, `钱塘156-1`）

**`河道功能区-基础信息` sheet**：由输入表自动生成的结构化信息（每个功能区的干流+支流各一行）

### 你想要的效果

在 Streamlit 应用中**直接上传这种带支流的 xlsx 文件**，自动完成：
1. 解析输入表中的支流信息（支流数量、名称、长度、汇入位置、浓度）
2. 解析逐日流量中干流和支流的分列流量
3. 按 VBA 的逻辑进行**分段计算**：干流被支流汇入点切分 → 逐段计算 → 混合点混合 → 继续下一段
4. 输出与 VBA 一致的结果（逐日纳污能力、逐月纳污能力、纳污能力过程、纳污能力结果）

## 二、改造方案

### 2.1 数据结构扩展（`calc_core.py`）

新增支流数据结构：
```python
@dataclass
class Branch:
    """支流信息"""
    name: str              # 支流名称（如 钱塘156-1）
    length: float          # 支流长度 L (m)
    join_position: float   # 汇入干流的位置 (m)
    C0: float              # 支流入口浓度 (mg/L)
```

扩展 `Zone`：
```python
@dataclass
class Zone:
    # ... 现有字段 ...
    main_name: str              # 干流名称（如 钱塘156-0）
    branches: List[Branch]      # 支流列表
```

### 2.2 核心计算改造（`calc_core.py`）

新增 `calc_zone_segments()` 函数，复刻 VBA 的 `CalcZoneSegments` 逻辑：

```
输入：功能区参数 + 当日干流流量 + 各支流流量
处理：
  1. 按汇入位置排序支流
  2. 遍历每个支流汇入点：
     a. 计算当前位置→汇入点的干流段纳污能力
     b. 计算支流出流浓度（如支流有长度）
     c. 混合点：Q_mix = Q干 + Q支, C_mix = (Q干×C干 + Q支×C支出) / Q_mix
  3. 计算最后一段干流（最后汇入点→终点）
  4. 汇总：只累加干流段的 W
输出：各段详情 + 功能区总纳污能力
```

改造 `calc_monthly_capacity()`：
- 逐日逐功能区调用 `calc_zone_segments()`
- 传入当日各河流（干流+支流）的流量

### 2.3 输入解析改造（新增 `xlsx_parser.py`）

直接解析 `河道功能区-输入` sheet 的竖向布局：
- 读取行1~11的基础参数
- 读取行12+的支流信息（每条支流5行）
- 构建 `Zone` 对象列表（含 `branches`）

解析 `逐日流量` sheet：
- 识别干流列（`xxx-0`）和支流列（`xxx-1`, `xxx-2`...）
- 按功能区分组

### 2.4 Streamlit 界面改造（`web_app.py`）

- 上传后自动识别输入格式（新格式 vs 旧格式，向后兼容）
- 数据预览增加支流信息展示
- 结果增加两个新 tab：
  - **纳污能力过程**：展示每个功能区的分段计算详情（干流段、支流、混合点、小计）
  - **纳污能力结果**：只展示干流段和汇总

### 2.5 输出 xlsx 格式

与 VBA 输出一致，包含以下 sheet：
1. 逐日纳污能力（方案N）
2. 逐月纳污能力（方案N）
3. 纳污能力过程（方案N）— **新增**
4. 纳污能力结果（方案N）— **新增**

## 三、待确认问题

1. **是否需要支持多方案？** 输入表有"方案个数"字段，VBA 支持多方案。Streamlit 是否也需要？
2. **旧格式兼容**：是否仍需支持不含支流的旧输入格式？（建议保留兼容）
3. **逐日纳污能力 sheet**：VBA 输出了逐日结果（23000+行），Streamlit 是否也需要输出？还是只输出逐月？
4. **支流自身的纳污能力**：VBA 中支流的 W 会展示但不计入汇总，Streamlit 是否也这样处理？
